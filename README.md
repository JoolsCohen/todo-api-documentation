# Документация для учебного API "To-Do List"

Это учебный проект по созданию технической документации для REST API с использованием OpenAPI (Swagger) и Markdown.

## О проекте

Данный репозиторий содержит полную документацию к гипотетическому API для управления списком задач. API поддерживает основные CRUD-операции (Create, Read, Update, Delete).

**Цель проекта:** продемонстрировать навыки технического письма, работы со стандартом OpenAPI и организации документации.

## Интерактивная документация

Спецификация API написана в формате OpenAPI 3.0.
- **[Просмотреть документацию в Swagger UI](https://editor.swagger.io/?url=https://raw.githubusercontent.com/ВАШ_ЛОГИН_GITHUB/НАЗВАНИЕ_РЕПОЗИТОРИЯ/main/openapi.yaml)**

## Быстрый старт

### Базовый URL
Все эндпоинты API доступны по базовому адресу:
```
https://api.example-todo.com/v1
```

### Пример запроса

Получить список всех задач:

```bash
curl -X GET "https://api.example-todo.com/v1/tasks" \
  -H "Content-Type: application/json"
```

**Пример успешного ответа (200 OK):**

```json
[
  {
    "id": 1,
    "title": "Купить молоко",
    "description": "Обезжиренное, 1 литр",
    "completed": false,
    "createdAt": "2025-08-25T10:30:00Z"
  }
]
```

## 📋 Эндпоинты API

| Метод | Эндпоинт | Описание |
| :--- | :--- | :--- |
| `GET` | `/tasks` | Получить список всех задач |
| `POST` | `/tasks` | Создать новую задачу |
| `GET` | `/tasks/{id}` | Получить задачу по ID |
| `PUT` | `/tasks/{id}` | Обновить задачу |
| `DELETE` | `/tasks/{id}` | Удалить задачу |
